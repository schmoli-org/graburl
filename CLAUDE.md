# GrabURL

Web extension (Safari + Chrome + Firefox) that copies the active tab URL to the clipboard. Single-button UX: click the toolbar icon (or press `Command+Shift+C` on Safari, `Alt+Shift+C` on Chrome/Firefox) ŌåÆ URL is copied ŌåÆ popup shows "Copied URL" and closes automatically.

## Repo structure

| Path | Purpose |
|---|---|
| `web-extension/` | **Canonical** extension source ŌĆö edit here |
| `GrabURL/GrabURL Extension/Resources/` | Xcode copy of web-extension files ŌĆö must stay in sync |
| `GrabURL/GrabURL/Resources/` | macOS companion app UI (enable/disable extension) |

## Docs and plans

All design docs ŌĆö specs, plans, ADRs, scratch notes, internal runbooks ŌĆö live in the private submodule. The public tree carries no planning or design material.

| Path | What goes here | Visibility |
|---|---|---|
| `docs/private/` | All specs, plans, ADRs, working notes, internal runbooks | Private submodule |
| `.claude/plans/` | Claude Code session plans | Gitignored, ephemeral |

**Rule:** Never commit plans, specs, design docs, or working notes to the public tree. They belong in `docs/private/`. The public repo holds only code, build/release config, and standard OSS files (`README`, `LICENSE`, `CLAUDE.md`).

**Submodule note:** `docs/private/` is a git submodule. If it appears empty after a fresh clone, run `git submodule update --init`.

## Critical conventions

1. **Always update both `web-extension/` AND `GrabURL/GrabURL Extension/Resources/`** when changing any extension file. The Xcode build uses the `GrabURL Extension` copy. They are plain file copies, not symlinks.
2. `url-copy.js` is a plain IIFE ŌĆö no ES modules, no import/export. It exports via `globalThis.CopyUrlExtension`.
3. The Chrome and Firefox packages are derived from `web-extension/` at build time by `scripts/build-chrome.mjs` and `scripts/build-firefox.mjs`, which patch the manifest per browser (generic description, `Alt+Shift+C` shortcut ŌĆö both Chrome and Firefox reserve `Cmd/Ctrl+Shift+C` for DevTools). The Firefox build additionally injects a `browser_specific_settings.gecko` block (add-on ID, `strict_min_version`, `data_collection_permissions: none`) required for AMO signing. Never edit `dist/`; it is regenerated on every build.

## Common commands

A `Makefile` wraps every routine task. **When telling the user to run something, give the `make` target, not the underlying `pnpm`/script command** (e.g. `make web`, not `cd web && pnpm dev`). Run `make help` to discover available targets; treat the `Makefile` as the source of truth.

| File | Role |
|---|---|

| `web/src/layouts/Layout.astro` | Global CSS variables and font for the landing page |
| `web/src/pages/index.astro` | Landing page content and component styles |
